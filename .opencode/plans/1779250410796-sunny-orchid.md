# Plan: Resolve 500 Internal Server Error on Production for Contact Form Submission

## 1. Problem Analysis
When the contact form is submitted on production, a `500 (Internal Server Error)` is returned. In a Laravel application, a 500 error during contact form submission typically has two primary root causes:
1. **Mail Dispatch Failure**: The controller attempts to send the email synchronously. If the production mail server (SMTP, SES, etc.) is not configured, or if the credentials in the production `.env` are invalid/missing, the mailer throws a `TransportException`, halting the execution and returning a 500 status.
2. **Database Migration Missing**: If database migrations were not executed on the production server, the `inquiries` table will not exist, causing a `QueryException` when `Inquiry::create($validated)` is executed.

---

## 2. Proposed Solution & Implementation Plan
To fix this robustly and prevent future disruptions, we will implement the following steps:

### Step 1: Wrap Mail Dispatch in a Try-Catch Block (Defensive Programming)
We should ensure that even if the mail server experiences downtime or is misconfigured, the customer's inquiry is still safely stored in the database, and the user receives a success response.
- **File**: `app/Http/Controllers/ContactController.php`
- **Change**: Wrap `Mail::to(...)->send(...)` in a `try-catch` block, catching any `Throwable`, reporting it to Laravel's logging system, and allowing the request to proceed successfully.

### Step 2: Diagnostic Guidance for Production
Provide the user with clear instructions to check and configure their production environment (`.env` on Hostinger) and ensure migrations are run:
1. **Check Mail Configuration**: Ensure valid SMTP settings are defined in `.env` (e.g., `MAIL_MAILER=smtp`, `MAIL_HOST`, `MAIL_PORT`, `MAIL_USERNAME`, `MAIL_PASSWORD`, `MAIL_ENCRYPTION`).
2. **Verify Database Migrations**: Ensure `php artisan migrate` has been executed on the production database.
3. **Verify App Logs**: Look at the Laravel error logs on production (`storage/logs/laravel.log`) to confirm the exact exception trace.

---

## 3. Verification Plan
- **Unit/Feature Tests**: Run local tests (`php artisan test`) to ensure the contact form flow and the `InquiryReceived` mail sending tests still pass cleanly with the try-catch block.
- **Simulate Mail Failures**: Write a test case (or temporarily simulate in the controller) where mail sending throws an exception, and verify that the database record is still created and the request succeeds.
